# <a name="common-power-bi-workspace-collection-scenarios"></a>Vanliga scenarier för Power BI-Arbetsytesamling

Power BI-Arbetsytesamlingar fokuserar på att exponera merparten av befintliga Power BI-API-funktioner som en del av en Azure-tjänst som du kan utveckla ditt program med.  Dessutom kan kan du etablera, utveckla och distribuera resurser som krävs och Power BI-innehåll. Mer information finns i [Kom igång med Microsoft Power BI-Arbetsytesamlingar](get-started.md).

> [!IMPORTANT]
> Power BI-arbetsytesamlingar fasas ut och är tillgänglig till juni 2018 eller det som anges i ditt avtal. Du uppmanas att planera migreringen till Power BI Embedded för att undvika avbrott i programmet. Information om hur du migrerar dina data till Power BI Embedded finns i [Migrera Power BI-arbetsytesamlingar till Power BI Embedded](https://powerbi.microsoft.com/documentation/powerbi-developer-migrate-from-powerbi-embedded/).

Här följer några scenarier för att använda **Power BI-Arbetsytesamlingar** i dina program.

* Helt interaktiva rapporter som redigerats i Power BI Desktop kan bäddas in i ditt program med hjälp av en ny modell för autentisering med enhetstoken i programmet.
* Oberoende programvaruleverantörer och kunder som bygger kundorienterade programmen kan använda den **Power BI-Arbetsytesamling** -tjänsten och Power BI SDK, för att bädda in interaktiva rapporter. Läs hur genom att läsa [Kom igång med Microsoft Power BI-Arbetsytesamlingar](get-started.md).
* Du kan använda framework för Power BI-visualisering som utvecklare för att skapa anpassade visualiseringar som kan användas i dina egna appar. Se [utöka Power BI med anpassade visuella objekt](https://powerbi.microsoft.com/custom-visuals/).
