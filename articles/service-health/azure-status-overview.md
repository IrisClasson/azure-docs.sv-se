# <a name="azure-status-overview"></a>Översikt över Azure-status

[Azure-status](https://status.azure.com/status/) ger dig en global vy över hälsotillståndet för Azure-tjänster och azure-regioner. Med Azure-status kan du få information om tjänstens tillgänglighet. Azure-status är tillgänglig för alla för alla att visa alla tjänster som rapporterar deras tjänsthälsa, samt incidenter med omfattande effekt. Om du är en aktuell Azure-användare rekommenderar vi dock starkt att du använder den anpassade upplevelsen i [Azure Service Health](https://aka.ms/azureservicehealth). Azure Service Health omfattar alla avbrott, kommande planerade underhållsaktiviteter och servicerekommendationer.

![Azure-statussida](./media/azure-status-overview/azure-status.PNG)

## <a name="azure-status-updates"></a>Azure-statusuppdateringar

Azure-statussidan uppdateras i realtid när hälsotillståndet för Azure-tjänster ändras. Om du lämnar azure-statussidan öppen kan du styra hur långt sidan uppdateras med nya data. Högst upp kan du se när sidan senast uppdaterades.

![Uppdatering av Azure-status](./media/azure-status-overview/update.PNG)

## <a name="azure-status-history"></a>Statushistorik för Azure

Medan Azure-statussidan alltid visar den senaste hälsoinformationen kan du visa äldre händelser med hjälp av [sidan Azure-statushistorik](https://status.azure.com/status/history/).

Från och med händelser som publicerades i november 2019 visar sidan statushistorik händelser upp till 365 dagar tidigare, inklusive en preliminär grundorsak, begränsning och nästa steg för Azure.

## <a name="rss-feed"></a>RSS-flöde

Azure-status ger också [en RSS-feed](https://status.azure.com/status/feed/) med ändringar i hälsotillståndet för Azure-tjänster som du kan prenumerera på.
