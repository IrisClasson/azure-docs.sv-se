# <a name="security-alerts---a-reference-guide"></a>Säkerhetsvarningar - en referensguide

I den här artikeln visas de säkerhetsaviseringar som kan visas i Azure Security Centers hotskyddsmodul. Aviseringarna som visas i din miljö beror på vilka resurser och tjänster du skyddar samt din anpassade konfiguration.

Mer information om hur du svarar på dessa aviseringar finns i [Hantera och svara på säkerhetsaviseringar i Azure Security Center](security-center-managing-and-responding-alerts.md).

Mer information om hur du exporterar aviseringar (och rekommendationer) finns [i Exportera säkerhetsaviseringar och rekommendationer (förhandsversion).](continuous-export.md)

Under aviseringar tabellerna är en tabell som beskriver Azure Security Center kill kedja som används för att kategorisera avsikter dessa aviseringar.

## <a name="intentions"></a>Avsikter

Förstå avsikten med en attack kan hjälpa dig att undersöka och rapportera händelsen lättare. Azure Security Center-aviseringar innehåller "avsiktsfältet" för att hjälpa till med dessa ansträngningar.

Den serie steg som beskriver utvecklingen av en cyberattack från spaning till dataexfiltration kallas ofta för en "kill chain". 

Security Centers stödda döda kedja avsikter är baserade på [MITRE&trade; ATT&CK ram](https://attack.mitre.org/matrices/enterprise) och beskrivs i tabellen nedan.

|Avsikt|Beskrivning|
|------|-------|
|**Föreattack**|PreAttack kan vara antingen ett försök att komma åt en viss resurs oavsett ett ont uppsåt, eller ett misslyckat försök att få tillgång till ett målsystem för att samla in information före utnyttjande. Det här steget identifieras vanligtvis som ett försök, som kommer från utanför nätverket, att söka igenom målsystemet och identifiera en startpunkt.</br>Mer information om PreAttack-stadiet kan läsas på [MITRE:s sida](https://attack.mitre.org/matrices/pre/).|
|**InitialAccess (InitialAccess)**|InitialAccess är det stadium där en angripare lyckas få fotfäste på den attackerade resursen. Den här fasen är relevant för beräkningsvärdar och resurser som användarkonton, certifikat etc. Hotaktörer kommer ofta att kunna kontrollera resursen efter detta skede.|
|**Bevarande**|Persistens är alla åtkomst-, åtgärders- eller konfigurationsändringar till ett system som ger en hotaktör en beständig närvaro på det systemet. Hotaktörer måste ofta upprätthålla åtkomsten till system genom avbrott som omstarter av systemet, förlust av autentiseringsuppgifter eller andra fel som skulle kräva ett fjärråtkomstverktyg för att starta om eller tillhandahålla en alternativ bakdörr för att de ska få åtkomst.|
|**PrivilegiumScalation**|Eskalering av privilegier är resultatet av åtgärder som gör det möjligt för en motståndare att få en högre behörighetsnivå för ett system eller nätverk. Vissa verktyg eller åtgärder kräver en högre nivå av privilegium att arbeta och är sannolikt nödvändiga på många ställen under en operation. Användarkonton med behörighet att komma åt specifika system eller utföra specifika funktioner som krävs för att motståndare ska kunna uppnå sitt mål kan också betraktas som en eskalering av privilegier.|
|**FörsvarEvasion**|Defense skatteflykt består av tekniker en motståndare kan använda för att undgå upptäckt eller undvika andra försvar. Ibland är dessa åtgärder samma som (eller variationer av) tekniker i andra kategorier som har den extra fördelen att undergräva ett visst försvar eller begränsning.|
|**AutentiseringsuppgifterAccess**|Åtkomst till autentiseringsuppgifter representerar tekniker som resulterar i åtkomst till eller kontroll över system-, domän- eller tjänstautentiseringsuppgifter som används i en företagsmiljö. Motståndare kommer sannolikt att försöka få legitima autentiseringsuppgifter från användare eller administratörskonton (lokal systemadministratör eller domänanvändare med administratörsbehörighet) att använda i nätverket. Med tillräcklig åtkomst i ett nätverk kan en motståndare skapa konton för senare användning i miljön.|
|**Identifiering**|Discovery består av tekniker som gör det möjligt för motståndaren att få kunskap om systemet och det interna nätverket. När motståndare får tillgång till ett nytt system måste de orientera sig till vad de nu har kontroll över och vilka fördelar som fungerar från det systemet ger deras nuvarande mål eller övergripande mål under intrånget. Operativsystemet ger många inbyggda verktyg som stöd i denna post-kompromiss informationsinsamling fas.|
|**LateralFlytt**|Lateral rörelse består av tekniker som gör det möjligt för en motståndare att komma åt och styra fjärrsystem i ett nätverk och kan, men inte nödvändigtvis, omfatta utförande av verktyg på fjärrsystem. Den laterala rörelse tekniker kan tillåta en motståndare att samla in information från ett system utan att behöva ytterligare verktyg, till exempel en fjärråtkomst verktyg. En motståndare kan använda laterala rörelser för många ändamål, inklusive fjärrkörning av verktyg, pivotering till ytterligare system, tillgång till specifik information eller filer, tillgång till ytterligare autentiseringsuppgifter eller för att orsaka effekt.|
|**Körnings-**|Körningstaktiken representerar tekniker som resulterar i utförande av motståndare-kontrollerad kod på ett lokalt eller avlägset system. Denna taktik används ofta i samband med laterala rörelser för att utöka tillgången till fjärrsystem i ett nätverk.|
|**Samling**|Samlingen består av tekniker som används för att identifiera och samla in information, till exempel känsliga filer, från ett målnätverk före exfiltration. Denna kategori omfattar även platser i ett system eller nätverk där motståndaren kan söka efter information som ska exfiltrates.|
|**Exfiltration (exfiltration)**|Exfiltration refererar till tekniker och attribut som resulterar eller hjälper motståndaren att ta bort filer och information från ett målnätverk. Denna kategori omfattar även platser i ett system eller nätverk där motståndaren kan söka efter information som ska exfiltrates.|
|**KommandoOchControl**|Kommando- och kontrolltaktiken representerar hur motståndare kommunicerar med system under deras kontroll inom ett målnätverk.|
|**Påverkan**|Påverkanshändelser försöker i första hand direkt minska tillgängligheten eller integriteten för ett system, en tjänst eller ett nätverk. inklusive manipulering av data för att påverka en affärs- eller operativ process. Detta skulle ofta hänvisa till tekniker som ransomware, defacement, datamanipulation och andra.|
