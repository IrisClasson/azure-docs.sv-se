# <a name="azure-cognitive-services-support-and-help-options"></a>Support- och hjälpalternativ för Azure Cognitive Services

Har du precis börjat utforska funktionerna i Azure Cognitive Services? Implementerar du en ny funktion i ditt program? Här är förslag på var du kan få hjälp som utvecklare.

> [!div class="checklist"]
> * Håll dig informerad om nya utvecklingar i *Azure Cognitive Services*.
> * Viktig information innehåller information för alla versioner.
> * Sök för att se om problemet har diskuterats av communityn eller om befintlig dokumentation redan finns.
> * Om du inte kan hitta ett tillfredsställande svar ställer du en fråga om *Stack Overflow*.
> * Om du hittar ett problem med något av exemplen på GitHub tar du upp ett *GitHub-problem.*
> * Sök efter en lösning i *UserVoice-forumet*.

## <a name="create-an-azure-support-request"></a>Skapa en Azure-supportbegäran

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

Utforska utbudet av [Azure-supportalternativ och välj den plan](https://azure.microsoft.com/support/plans) som bäst passar, oavsett om du är en utvecklare som precis har påbörjat din molnresa eller en stor organisation som distribuerar affärskritiska, strategiska program. Azure-kunder kan skapa och hantera supportärenden i Azure-portalen.

* [Azure-portal](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [Azure-portal för USA:s regering](https://portal.azure.us)

## <a name="post-a-question-on-stack-overflow"></a>Skicka en fråga om Stack Overflow

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

Om du vill ha omedelbara svar från communityn kan du se till Stack Overflow. Stack Overflow är den föredragna kanalen för utvecklingsrelaterade frågor. Det är där medlemmar i communityn och Microsoft-teammedlemmar är direkt involverade i att hjälpa dig att lösa dina problem.

Om du inte hittar ett svar på problemet via sökning skickar du en ny fråga till Stack Overflow. Använd någon av följande taggar när du formulerar frågan:

 - [Microsoft Cognitive](https://stackoverflow.com/questions/tagged/microsoft-cognitive)
 - [Azure Cognitive Services](https://stackoverflow.com/questions/tagged/azure-cognitive-services)

> [!TIP]
> Följande inlägg från Stack Overflow innehåller tips om hur du bildar frågor och lägger till källkod. Om du följer dessa riktlinjer kan du öka chanserna för communitymedlemmar att snabbt bedöma och svara på din fråga:
> * [Hur ställer jag en bra fråga?](https://stackoverflow.com/help/how-to-ask)
> * [Hur skapar man ett minimalt, reproducerbart exempel?](https://stackoverflow.com/help/minimal-reproducible-example)

### <a name="scoped-search"></a>Begränsad sökning

Om du vill ha snabbare resultat kan du begränsa sökningen till relevanta webbplatser som Stack Overflow, Microsoft docs eller GitHub-kodexempel. Använd `site:` frågeetsationen i din [favoritsökmotor:](https://bing.com)

```
{search keywords} site:stackoverflow.com
```

Var `{search keywords}` är sammanhanget för din sökning. Den här sökningen kommer att begränsa resultaten till Stack Overflow. Överväg att använda dessa begränsade sökningar på rätt sätt:

 - Stack spill:`site:stackoverflow.com`
 - Microsoft Dokument:`site:docs.microsoft.com`
 - GitHub-exempel:`site:github.com/azure-samples`

## <a name="create-a-github-issue"></a>Skapa ett GitHub-problem

<div class='icon is-large'>
    <img alt='GitHub' src='https://docs.microsoft.com/media/logos/logo_octokitty.svg'>
</div>

Exempel bokförs ofta som öppen källkod. Skapa ett *problem* i respektive GitHub-databas för frågor. Du kan skicka en pull begäran också! Letar du efter **Azure Samples** källkod, nedan är en lista över Cognitive Services exempel källkod:

* [Azure-exempel - Kognitiva tjänster](https://github.com/Azure-Samples?utf8=%E2%9C%93&q=cognitive&type=&language=)

Du kan skapa en felrapport, funktionsbegäran eller ställa en allmän fråga och dela bästa praxis. För felrapporter, följ den angivna mallen:

**Beskriv felet**

En tydlig och koncis beskrivning av vad felet är.

**Om du vill reproducera**

Åtgärder för att återskapa beteendet:
1. ...
2. ...

**Förväntat beteende**

En tydlig och koncis beskrivning av vad du förväntade dig att hända.

**Vilka Cognitive Services-erbjudanden använder du**

Beskriv det Cognitive Services-erbjudande som du försöker använda och ange information om vad du försöker uppnå.

**Plattforms-, operativsystem- och programmeringsspråk**

 - OS: [t.ex. Windows, Linux, Android, iOS, ...] - var specifik
 - Hårdvara - x64, x86, ARM, ...
 - webbläsare, t.ex.

**Kontext**

 - Felmeddelanden, logginformation, stackspårning, ...
 - Om du rapporterar ett fel för en viss tjänstinteraktion rapporterar du SessionId och tid (inkl. tidszon) för de rapporterade incidenterna. SessionId rapporteras i alla call-backs/händelser som du får.
 - All annan ytterligare information
