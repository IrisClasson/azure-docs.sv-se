# <a name="devtest-labs-in-the-enterprise"></a>DevTest Labs i företaget
Företag snabbt går över till molnet grund av [fördelar](/azure/architecture/cloud-adoption/business-strategy/cloud-migration-business-case) som omfattar rörlighet, flexibilitet och ekonomin. De första stegen är ofta utveckling och testa arbetsbelastningar. Azure DevTest Labs ger [funktioner](devtest-lab-concepts.md) som dra enterprise och support [nyckeln utveckling och testning](devtest-lab-guidance-get-started.md).

Vanliga problem för företag som migrerar arbetsbelastningar till molnet är:

- [Skydda resurser för utveckling och testning](devtest-lab-guidance-governance-policy-compliance.md)
- [Hantera och förstå kostnaderna](devtest-lab-guidance-governance-cost-ownership.md)
- Aktiverar Självbetjäning för utvecklare utan att kompromissa med företagssäkerhet och efterlevnad
- Automatisera och utöka DevTest Labs för att täcka ytterligare scenarier
- [Skala en DevTest Labs-baserad lösning till tusentals resurser](devtest-lab-guidance-scale.md)
- [Storskaliga distributioner av DevTest Labs](devtest-lab-guidance-orchestrate-implementation.md)
- [Komma igång med ett konceptbevis](devtest-lab-guidance-orchestrate-implementation.md)

## <a name="intended-audience"></a>Målgrupp
Den här dokumentationen avser företagens IT planerare, arkitekter och chefer som ansvarar för att upprätta och granska distributioner och övervaka åtgärder. De här artiklarna betona de övergripande processen och rekommenderade designprinciperna. Målet är att främja en miljö med säkra och stabila utveckling/testning, som slutligen styr användningen av Azure DevTest Labs inom en organisation.

## <a name="enterprise-customers"></a>Enterprise-kunder

Många aktuella DevTest Labs-företagskunder använda kunna labb för utveckling och för att testa arbetsbelastningar i sina organisationer. [Läs mer](https://azure.microsoft.com/case-studies/?term=DevTest+labs).
