# <a name="request-public-transit-data-using-the-azure-maps-mobility-service"></a>Begär offentlig överföring data med hjälp av Azure Maps Mobilitetstjänsten 

Den här artikeln visar hur du använder Azure Maps [Mobilitetstjänsten](https://aka.ms/AzureMapsMobilityService) för att begära offentliga överföring data, inklusive stoppas, dirigera information och reser mycket tid uppskattningar.

I den här artikeln som du får lära dig hur du:

* Hämta en metro området ID med hjälp av den [hämta API för appar i Metro-området](https://aka.ms/AzureMapsMobilityMetro)
* Begär i närheten överföring slutar använda sig av [hämta i närheten överföring](https://aka.ms/AzureMapsMobilityNearbyTransit) service.
* Fråga [hämta överföring vägar API](https://aka.ms/AzureMapsMobilityTransitRoute) att planera en väg med hjälp av offentliga överföring.
* Begär överföring vägen geometri och detaljerad schemat för en väg med hjälp av den [hämta överföring resan API](https://aka.ms/ https://azure.microsoft.com/services/azure-maps/).


## <a name="prerequisites"></a>Nödvändiga komponenter

Om du vill göra något anrop till den offentliga Azure Maps-överföringen API: er, behöver du en Maps-konto och nyckel. Information om att skapa ett konto och hämta en nyckel finns i [så här hanterar du ditt Azure Maps-konto och dina nycklar](how-to-manage-account-keys.md).

Den här artikeln används den [Postman-appen](https://www.getpostman.com/apps) att skapa REST-anrop. Du kan använda alla API-utvecklingsmiljö du föredrar.


## <a name="get-a-metro-area-id"></a>Hämta ett metro området-ID

[Hämta API för appar i Metro-området](https://aka.ms/AzureMapsMobilityMetro) kan du begära metro områden där Mobilitetstjänsten för Azure Maps är tillgänglig. Svaret innehålla information som `metroId`, `metroName` och en representation av metro området geometri i GeoJSON-format.

Vi gör en begäran att hämta appar i Metro-området för Seattle Tacoma metro-ID. Om du vill begära ID för ett metro område, gör du följande:
