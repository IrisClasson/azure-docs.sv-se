# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Vilka verktyg ingår i Virtual Machine för Azure Data Science?

Den virtuella datorn för datavetenskap är det enklaste sättet att utforska data och göra maskininlärning i molnet. Data Science Virtuella datorer är förkonfigurerade med hela operativsystemet, säkerhetskorrigeringar, drivrutiner och populär datavetenskap och utveckling programvara. Du kan välja maskinvarumiljö, allt från billigare CPU-centrerade datorer till mycket kraftfulla datorer med flera GPU:er, NVMe-lagring och stora mängder minne. För datorer med GPU:er installeras alla drivrutiner, alla maskininlärningsramverk är versionsmatchade för GPU-kompatibilitet och acceleration är aktiverat i all programvara som stöder GPU:er.

Den virtuella datavetenskapens virtuella dator levereras med de mest användbara datavetenskapsverktygen förinstallerade:

## <a name="how-to-choosecondaenvironment"></a>Hur man väljer conda miljö  

> [!NOTE]
> Det finns bryta ändringar när du byter till den nya versionen av Data Science Virtual Machine från den gamla versionen. 

**Byta från Windows Server 2016 till Windows Server 2019** </br> 

* Conda-miljön är inställd på python37_default. Vi stöder inte python2. Om du använde tensorflow eller pytorch i python conda-miljön använder du py37_tensorflow respektive py37_pytorch. Om du bara använde AzureML azureml_py36_automl </br>

* Om du använde tensorflow eller pytorch i din AzureML conda-miljö använder du azureml_py36_tensorflow respektive azureml_py36_pytorch. </br>

**Byta från Ubuntu 16,04 till Ubuntu 18,04** </br>

* Om du använde tensorflow eller pytorch i din azureml_py36 conda miljö använd azureml_py36_tensorflow eller azureml_py36_pytorch respektive. </br>

* Om du använde tensorflow eller pytorch i din py37 conda miljö använd py37_tensorflow eller py37_pytorch respektive. </br>

* Annars använder du python37_default eller om du bara använde azureml_py36 använda azureml_py36_automl </br>
