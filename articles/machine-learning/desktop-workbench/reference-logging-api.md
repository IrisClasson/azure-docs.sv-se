# <a name="logging-api-reference"></a>Loggning av API-referens

[!INCLUDE [workbench-deprecated](../../../includes/aml-deprecating-preview-2017.md)] 

Azure ML loggning-biblioteket gör att programmet kan skapa mått och filer som spåras av tjänsten historik för senare analys. 

## <a name="uploading-metrics"></a>Ladda upp mått

```python
# import logging API package
from azureml.logging import get_azureml_logger

# initialize a logger object
logger = get_azureml_logger()

# log "scalar" metrics
logger.log("simple integer value", 7)
logger.log("simple float value", 3.141592)
logger.log("simple string value", "this is a string metric")

# log a list of numerical values. 
# this automatically creates a chart in the Run History details page
logger.log("chart data points", [1, 3, 5, 10, 6, 4])
```

Som standard skickas alla mått asynkront så att överföringen inte hindra programkörningen. Detta kan orsaka skrivordning problem när flera mått skickas i gränsfall. Ett exempel på detta är två mått som är inloggad på samma gång, men av någon anledning som du vill använda deras exakta ordning bevaras. Ett annat fall är när måttet måste spåras innan som körde kod som du känner till potentiellt misslyckas snabbt. I båda fallen lösningen är att _vänta_ tills måttet loggas fullständigt innan du fortsätter:

```python
# blocking call
logger.log("my metric 1", 1).wait()
logger.log("my metric 2", 2).wait()
```

## <a name="consuming-metrics"></a>Använda mått

Mått som lagras av tjänsten historik och knutna till körningen som producerade dem. Både Körningshistorik fliken och CLI-kommandot nedan kan du hämta dem (och artefakter nedan) när en körning har slutförts.

```azurecli
# show the last run
$ az ml history last

# list all past runs
$ az ml history list 

# show a paritcular run
$ az ml history info -r <runid>
```

## <a name="artifacts-files"></a>Artefakter (filer)

Förutom mätvärden används AzureML att spåra filer samt. Som standard alla filerna som skrivits till den `outputs` mappen i förhållande till arbetskatalogen för programmets (projektmappen i beräkningskontexten) överförs till tjänsten historik och spåras för senare analys. Teamindela är att den enskilda filstorleken måste vara mindre än 512 MB.


```Python
# Log content as an artifact
logger.upload("artifact/path", "This should be the contents of artifact/path in the service")
```

## <a name="consuming-artifacts"></a>Förbrukar artefakter

Om du vill få en artefakt som har spårats kan användare använda fliken Körningshistorik för en viss körning till **hämta** eller **befordra** artefakt eller Använd den nedan CLI-kommandon för att uppnå samma effekt.

```azurecli
# show all artifacts generated by a run
$ az ml history info -r <runid> -a <artifact/path>

# promote a particular artifact
$ az ml history promote -r <runid> -ap <artifact/prefix> -n <name of asset to create>
```
