# <a name="azure-vfxt-for-azure-whitepapers-and-other-resources"></a>Azure vFXT för Azure-faktablad och andra resurser

I den här artikeln visas fallstudier, teknikvitpapper, infografik och andra resurser som kan hjälpa dig att förstå hur Avere vFXT för Azure passar in i arbetsflödet.

## <a name="deploy-agisoft-photoscan-on-azure"></a>Distribuera Agisoft PhotoScan på Azure

**Länk:** [Distribuera Agisoft PhotoScan på Azure med Avere vFXT för Azure eller BeeGFS](https://azure.microsoft.com/mediahandler/files/resourcefiles/deploy-agisoft-photoscan-on-azure-with-azere-vfxt-for-azure-or-beegfs/AgiSoft%20PhotoScan%20on%20Azure%20using%20Avere%20vFXT%20or%20BeeGFS.pdf)

Den här guiden ger steg-för-steg-vägledning för installation av Agisoft PhotoScan fotogrammetriprogramvara som backas upp av antingen Avere vFXT-lagring eller BeeGFS parallellfilsystem. Författaren Paulo Marques da Costa från AzureCAT visar hur du konfigurerar PhotoScan på virtuella Azure-datorer (VIRTUELLA datorer). Högpresterande lagring påskyndar bearbetningstiden, och resultaten av hans benchmark-tester ingår. Den här miljön kan skalas upp och ned efter behov och stöder terabyte lagringsutrymme utan att offra prestanda.

## <a name="datasheet-avere-vfxt-for-azure"></a>Datablad: Avere vFXT för Azure

Den här tvåsidiga briefen har grundläggande information och diagram som förklarar hur du använder Avere vFXT för Azure i flera användningsfallsscenarier.

## <a name="hybrid-cloud-for-file-based-workloads"></a>Hybridmoln för filbaserade arbetsbelastningar

**Länk:** [Infographic: Skapa ett hybridmoln för filbaserade arbetsbelastningar](https://azure.microsoft.com/resources/building-a-hybrid-cloud-for-file-based-hpc-workloads/)

Den här bilden visar komponenter och layouter för att använda Avere vFXT för Azure för att lägga till molninfrastruktur till ditt befintliga system. Använd Avere vFXT för Azure för att öka datorkapaciteten med din befintliga lokala infrastruktur, eller flytta data och databehandling till molnet på tillfällig eller permanent basis.
