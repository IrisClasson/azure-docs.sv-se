# <a name="azure-functions-and-serverless-platform-security"></a>Azure Functions och Server lös plattforms säkerhet
## <a name="abstract"></a>Abstrakt
De flesta företag behöver en stor mängd resurser och tid för att hantera servrar, vilket ökar kostnaderna. Om företag kan använda färre resurser för att hantera servrar, kan de fokusera på att skapa fantastiska program.  

Med data behandling utan server kan du göra det, eftersom den infrastruktur du behöver för att köra och skala dina appar hanteras åt dig. Databearbetning utan server är abstraktionen av servrar, infrastruktur och operativsystem. Data behandling utan server styrs av reaktionen på händelser och utlösare, som alla sker i nära real tid – i molnet. 

Som en fullständigt hanterad tjänst är Server hantering och kapacitets planering osynligt för utvecklaren. Det serverbaserade ramverket hjälper dig att utveckla och distribuera program utan server med hjälp av Azure Functions. Det är ett kommando rads gränssnitt (CLI) som erbjuder struktur och automatisering för att hjälpa dig att bygga avancerade, händelse drivna, Server fria arkitekturer som består av funktioner och händelser. En Azure-funktion är en oberoende distributions enhet, t. ex. en mikrotjänst. Det är bara kod, som distribueras i molnet, som oftast skrivs för att utföra ett enda jobb.

Trots fördelarna har Server lös säkerhet sina egna riskfaktorer för att hantera. Metoden utan server ger inga nya säkerhets problem, men det krävs en metod för befintliga säkerhets problem. Detta white paper fokuserar på följande säkerhets frågor: 
* Fördelar med en plattform utan Server
* Säkerhets problem i data behandling utan Server
* Viktiga säkerhets problem och begränsningar i Azures kontext
* Skydda Microsofts plattform utan Server
