# <a name="saas-application-publishing-prerequisites"></a>Publicera krav för SaaS-program

Den här artikeln beskrivs kraven för att publicera ett erbjudande med SaaS-program (App) på Azure Marketplace.  Om du inte redan har gjort det, kan du granska den [SaaS-program erbjuder publiceringsguide](../../marketplace-saas-applications-technical-publishing-guide.md). 


## <a name="publishing-prerequisites"></a>Krav för publicering

Om du vill publicera ett nytt erbjudande för SaaS-App, måste du uppfylla följande krav:
- Åtkomst till den [Cloud Partner Portal](https://cloudpartner.azure.com/).  Cloud Partner Portal ger rollbaserad åtkomst, vilket innebär att flera personer att samarbeta för att publicera ett erbjudande. 
- Innan ett erbjudande kan publiceras åt ett utgivarkonto, en av enskilda personer med rollen ägare måste du samtycker till att följa den [användningsvillkor](https://azure.microsoft.com/support/legal/website-terms-of-use/), [Microsoft Privacy Statement](https://www.microsoft.com/privacystatement/default.aspx), och [ Microsoft Azure Certified-programavtalet](https://azure.microsoft.com/support/legal/marketplace/certified-program-agreement/).
- Allt innehåll som icke-engelska måste lämnas in tillsammans med en engelsk språkversion. Det här innehållet innehåller Storefront text, dokument, skärmbilder, användningsvillkor och sekretesspolicy. Det går att ange en användbar länk-URL till ett program som inte är på engelska. 
- Ha dina metadata som är redo att användas. I följande lista (ofullständig) visas ett exempel på dessa metadata:
  - En rubrik
  - En beskrivning (i HTML-format)
  - En logotyp (i PNG-format) och fasta bildstorleken i den här: 40 x 40 bildpunkter, 90 x 90 bildpunkter, 115 x 115 bildpunkter och 255 x 115 bildpunkter.
  - En *användningsvillkor* och *sekretesspolicy* uttryck
  - Dokumentation om Application
  - Supportkontakter


## <a name="next-steps"></a>Nästa steg

När du har uppfyllt dessa krav, kan du [skapa SaaS-erbjudande](./cpp-create-offer.md).
