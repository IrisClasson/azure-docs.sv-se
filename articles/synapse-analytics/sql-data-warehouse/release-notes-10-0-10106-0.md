# <a name="azure-synapse-analytics-release-notes"></a>Viktig information om Azure Synapse Analytics

Den här artikeln sammanfattar de nya funktionerna och förbättringarna i de senaste versionerna av [Synapse SQL](sql-data-warehouse-overview-what-is.md) i Azure Synapse. I artikeln visas också anmärkningsvärda innehållsuppdateringar som inte är direkt relaterade till versionen men som publiceras inom samma tidsram. Förbättringar av andra Azure-tjänster finns i [Tjänstuppdateringar](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Kontrollera din Azure Synapse-version

När nya funktioner distribueras till alla regioner kontrollerar du den version som distribueras till din instans och de senaste versionsinformationerna för funktionstillgänglighet. Om du vill kontrollera versionen ansluter du till SQL-poolen via `SELECT @@VERSION;` SQL Server Management Studio (SSMS) och kör för att returnera den aktuella versionen.

Använd den version som identifierats för att bekräfta vilken version som har tillämpats på din SQL-pool. Datumet i utdata identifierar månaden för den version som tillämpas på DIN SQL-pool.

> [!NOTE]
> Produktnamnet som returneras@VERSION av SELECT @ ändras från Microsoft Azure SQL Data Warehouse till Azure Synapse Analytics. Vi kommer att skicka ett förhandsmeddelande innan ändringen görs. Den här ändringen är relevant för kunder som tolkar@VERSION produktnamnet från resultatet av SELECT @ i sin programkod. För att undvika ändringar av programkod på grund av produktomprofilering, använd dessa kommandon för att fråga SERVERPROPERTY för databasens produktnamn och version: För att returnera versionsnummer XX. X.XXXXX. X (utan produktnamn) använd det här kommandot:
>
> ```sql
> SELECT SERVERPROPERTY('ProductVersion')
>
> --To return engine edition, use this command that returns 6 for Azure Synapse Analytics (Formerly SQL Data Warehouse):
>
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Med ett enda klick på en knapp kan du nu konfigurera diagnostikloggar för historiska felsökningsfunktioner för frågeprestanda med [Log Analytics](../../azure-monitor/log-query/log-query-overview.md?toc=/azure/synapse-analytics/sql-data-warehouse/toc.json&bc=/azure/synapse-analytics/sql-data-warehouse/breadcrumb/toc.json). Azure Monitor-diagnostikloggar stöder anpassningsbara kvarhållningsperioder genom att spara loggarna till ett lagringskonto i granskningssyfte, möjligheten att strömma loggar till händelsehubbar nära telemetriinsikter i realtid och möjligheten att analysera loggar med hjälp av Logganalys med loggfrågor. Diagnostikloggar består av telemetrivyer i ditt informationslager som motsvarar de vanligaste prestandafelsöknings-DVS för SQL Analytics i Azure Synapse. |**Hantering av Columnstore-minne**|I takt med att antalet komprimerade kolumnarkivradgrupper ökar ökar det minne som krävs för att hantera de interna kolumnsegmentmetadata för dessa radgrupper. Förbättringar har gjorts i den här versionen för att optimera storleken på de interna metadata för dessa fall, vilket leder till förbättrad erfarenhet och prestanda för sådana frågor.|
