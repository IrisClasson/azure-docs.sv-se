# <a name="get-kubelet-logs-from-azure-kubernetes-service-aks-cluster-nodes"></a>Hämta kubelet-loggar från AKS-klusternoder (Azure Kubernetes Service)

Som en del av driften av ett AKS-kluster kan du behöva granska loggar för att felsöka ett problem. Inbyggd i Azure-portalen är möjligheten att visa loggar för [AKS-huvudkomponenter][aks-master-logs] eller [-behållare i ett AKS-kluster][azure-container-logs]. Ibland kan du behöva hämta *kubelet-loggar* från en AKS-nod för felsökning.

Den här artikeln visar `journalctl` hur du kan använda för att visa *kubeletloggarna* på en AKS-nod.

## <a name="before-you-begin"></a>Innan du börjar

Den här artikeln förutsätter att du har ett befintligt AKS-kluster. Om du behöver ett AKS-kluster läser du SNABBSTARTen för AKS [med Azure CLI][aks-quickstart-cli] eller använder [Azure-portalen][aks-quickstart-portal].

## <a name="create-an-ssh-connection"></a>Skapa en SSH-anslutning

Skapa först en SSH-anslutning med den nod där du behöver visa *kubeletloggar.* Den här åtgärden beskrivs i [SSH i AKS-klusternoder (Azure Kubernetes Service)][aks-ssh] .

## <a name="get-kubelet-logs"></a>Hämta kubelet-loggar

När du har anslutit till noden kör du följande kommando för att hämta *kubeletloggarna:*

```console
sudo journalctl -u kubelet -o cat
```
