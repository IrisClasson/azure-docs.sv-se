# <a name="azure-active-directory-identity-protection-faq"></a>Azure Active Directory Identity Protection vanliga frågor och svar

Den här artikeln innehåller svar på vanliga frågor och svar om Azure Active Directory (Azure AD) Identity Protection. Mer information finns i [Azure Active Directory Identity Protection](../active-directory-identityprotection.md). 


## <a name="why-do-some-risk-events-have-closed-system-status"></a>Varför har vissa riskhändelser ”stängd (system)” status?

**A:** Dessa riskhändelser har identifierats av identitetsskydd och senare stängas på grund av händelser har inte längre vara riskfyllda. Dessa händelser räknas inte mot användarens risknivå. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>Måste jag vara global administratör för att använda Identity Protection i Azure-portalen?
**A:** Nej. Du kan antingen vara en Säkerhetsläsare, en säkerhetsadministratör eller Global administratör för att använda Identity Protection.

---

## <a name="how-do-i-get-identity-protection"></a>Hur får jag Identity Protection?

**A:** Se [komma igång med Azure Active Directory Premium](../fundamentals/active-directory-get-started-premium.md) efter ett svar på den här frågan.

---

## <a name="how-can-i-sort-users-in-users-flagged-for-risk"></a>Hur sorterar jag användare i ”användare som har flaggats för risk”?

**A:** Ladda ned användare som flaggats för risk genom att klicka på **hämta** i den **användare som har flaggats för risk** sidan. Du kan sedan sortera nedladdade data baserat på tillgängliga fält, inklusive senast uppdaterat (UTC).
