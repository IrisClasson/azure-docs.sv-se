# <a name="archived-release-notes"></a>Arkiverad viktig information

> [!IMPORTANT]  
> Linux är det enda operativsystemet som används med HDInsight version 3.4 och senare. Mer information finns i [artikeln om versions hantering i HDInsight](hdinsight-component-versioning.md).

## <a name="summary"></a>Sammanfattning

Azure HDInsight är en av de mest populära tjänsterna bland företagskunder för analys med Apache Hadoop med öppen källkod och Apache Spark på Azure.

## <a name="new-features"></a>Nya funktioner

De nya uppdateringarna och funktionerna hamnar i följande kategorier:

*  ***Uppdatera Hadoop och andra projekt med öppen källkod*** – förutom 1000 + fel korrigeringar över 20 + projekt med öppen källkod, innehåller den här uppdateringen en ny version av **Spark (2,3)** och **Kafka (1,0)** .

    a.  [**Nya funktioner i Apache Spark 2,3**](https://spark.apache.org/releases/spark-release-2-3-0.html)

    b.  [**Nya funktioner i Apache Kafka 1,0**](https://kafka.apache.org/downloads#1.0.0)

*  ***Uppdatering R Server 9,1 till Machine Learning Services 9,3*** – med den här versionen tillhandahåller vi data experter och tekniker med den bästa av öppen källkod som har utökats med algoritmer och driftsättning, som alla är tillgängliga i deras önskemål språk med hastigheten på Apache Spark. Den här versionen expanderar de funktioner som erbjuds i R Server med stöd för python, vilket leder till kluster namnet som ändras från R Server till ML-tjänster. 

*  ***Stöd för Azure Data Lake Storage Gen2*** – HDInsight kommer att stödja för hands versionen av Azure Data Lake Storage Gen2. I tillgängliga regioner kommer kunderna att kunna välja ett ADLS Gen2 konto som primär eller sekundär lagring för sina HDInsight-kluster.

*  ***HDInsight Enterprise Security Package-uppdateringar (*** för hands version) – (för hands version) [Virtual Network tjänst slut punkter](https://docs.microsoft.com/azure/virtual-network/virtual-network-service-endpoints-overview) stöd för Azure Blob Storage, ADLS gen1, Cosmos DB och Azure dB.

### <a name="mahout"></a>Mahout

I HDP-2.3. x och 2.4. x, i stället för att leverera en viss Apache-version av Mahout, synkroniserades vi med en viss revisions punkt på Apache Mahout trunk. Den här revisions punkten är efter 0.9.0-versionen, men före 0.10.0-versionen. Detta ger ett stort antal fel korrigeringar och funktions förbättringar i 0.9.0-versionen, men tillhandahåller en stabil version av Mahout-funktionerna innan den fullständiga konverteringen till nya Spark-baserade Mahout i 0.10.0.

Den ändrings punkt som valts för Mahout i HDP 2.3. x och 2.4. x är från grenen "Mahout-0,10. x" i Apache Mahout, från och med 19 december 2014, revisions 0f037cb03e77c096 i GitHub.

I HDP-2.5. x och 2.6. x tog vi bort biblioteket "Commons-httpclient" från Mahout eftersom vi visar det som ett föråldrat bibliotek med möjliga säkerhets problem och uppgraderat Hadoop-client i Mahout till version 2.7.3, samma version som används i HDP-2,5. Därför:

-   Tidigare kompilerade Mahout-jobb måste kompileras om i HDP-2,5-eller 2,6-miljön.

-   Det finns en liten risk att vissa Mahout-jobb kan stöta på "ClassNotFoundException" eller "Det gick inte att läsa in klass" fel relaterade till "org. apache. Commons. HttpClient", "net. java. dev. jets3t" eller relaterade prefix för klass namn. Om felen inträffar kan du fundera över om du vill installera de nödvändiga jar v7 manuellt i din classpath för jobbet, om risken för säkerhets problem i det föråldrade biblioteket är acceptabel i din miljö.

-   Det finns en ännu mindre möjlighet att vissa Mahout-jobb kan stöta på krascher i Mahouts HBase-client-kod anrop till de Hadoop-gemensamma biblioteken på grund av problem med binära kompatibilitetsproblem. Regrettably, det finns inget sätt att lösa det här problemet, förutom att återgå till HDP-2.4.2-versionen av Mahout, som kan ha säkerhets problem. Återigen bör det vara ovanligt, och det är osannolikt att det sker i en bestämd Mahout-programsvit.

  1. Hem kataloger för användare skapas inte på Head-noden 1. Som en lösning skapar du kataloger manuellt och ändrar ägarskapet till respektive användares UPN.

     ```bash
     chmod 751 /hdp 
     chmod –R 755 /hdp/apps
     ```

  Orsaken till det här felet är att den Tomcat-version som används i Ranger är\*Tomcat-7.0.7. Användning av JDK-1,7-konflikter med standardchiffer som finns i Tomcat\*-7.0.7.
    
  Du kan lösa det här problemet på två sätt:
    
  -   Uppdatera JDK som används i Ambari från JDK-1,7 till JDK-1,8 (se avsnittet [ändra JDK-versionen](https://docs.hortonworks.com/HDPDocuments/Ambari-2.2.0.0/bk_ambari_reference_guide/content/ch_changing_the_jdk_version_on_an_existing_cluster.html) i referens hand boken för Ambari).
    
  -   Om du vill fortsätta att stödja en JDK-1,7-miljö:
    
      1.  Lägg till egenskapen Ranger. Tomcat. Cipher i avsnittet Ranger-admin-site i din Ambari Ranger-konfiguration med värdet nedan:
            
          SSL\_-\_RSAMED\_RC4128\_MD5,SSL\_RSAMED\_RC4128 SHA,TLS\_RSA\_\_\_\_\_ MED\_AES\_128CBC\_SHA,\_SSL RSAMED3DES\_EDECBC\_SHA\_\_\_\_
        
      2.  Om din miljö har kon figurer ATS för Ranger-KMS lägger du till egenskapen Ranger. Tomcat. Cipher i theranger-KMS-site i din Ambari Ranger-konfiguration med värdet nedan:
            
          SSL\_-\_RSAMED\_RC4128\_MD5,SSL\_RSAMED\_RC4128 SHA,TLS\_RSA\_\_\_\_\_ MED\_AES\_128CBC\_SHA,\_SSL RSAMED3DES\_EDECBC\_SHA\_\_\_\_

  >De angivna värdena fungerar som exempel och är kanske inte indikationer på din miljö. Se till att du anger dessa egenskaper stämmer överens med hur din miljö har kon figurer ATS.

## <a name="deprecation"></a>Utfasning

-   **OMS-Portal:** Vi har tagit bort länken från HDInsight-resurs sidan som pekade på OMS-portalen. Azure Monitor loggar ursprungligen använde sin egen portal som heter OMS-portalen för att hantera konfigurationen och analysera insamlade data. Alla funktioner från den här portalen har flyttats till den Azure Portal där den kommer att fortsätta att utvecklas. HDInsight har föråldrat stödet för OMS-portalen. Kunder kommer att använda HDInsight Azure Monitor loggar integrering i Azure Portal.

-   **Spark 2,3**

    -   <https://spark.apache.org/releases/spark-release-2-3-0.html#deprecations>

## <a name="upgrading"></a>Uppgraderar

Alla dessa funktioner är tillgängliga i HDInsight 3,6. Du kan hämta den senaste versionen av Spark, Kafka och R Server (Machine Learning Services) genom att välja Spark, Kafka, ML Services-versionen när du [skapar ett HDInsight 3,6-kluster](https://docs.microsoft.com/azure/hdinsight/hdinsight-hadoop-provision-linux-clusters). Om du vill ha stöd för ADLS kan du välja lagrings typen ADLS som ett alternativ. Befintliga kluster uppgraderas inte automatiskt till dessa versioner.

Alla nya kluster som skapats efter juni 2018 får automatiskt de 1 000 + fel korrigeringarna i alla projekt med öppen källkod. Följ [den här](https://docs.microsoft.com/azure/hdinsight/hdinsight-upgrade-cluster) guiden för bästa praxis för att uppgradera till en nyare HDInsight-version.
