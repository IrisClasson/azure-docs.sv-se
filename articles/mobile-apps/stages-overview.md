# <a name="different-stages-in-mobile-application-development"></a>Olika steg i utvecklingen av mobila applikationer
Att bygga en mobil applikation innebär flera steg. Du kan enkelt dra nytta av Microsofts tjänster, verktyg och tekniker för att skapa högklassiga mobila program.

## <a name="app-development"></a>Apputveckling
Du kan skapa inbyggda klientprogram med en plattform med språk som Objective-C och Java. Du kan skapa plattformsoberoende appar med Xamarin, React Native och Unity. Du kan också skapa hybridappar med Cordova. Om du vill skapa klientprogram använder du ID:er och kodredigerare, till exempel Visual Studio, Visual Studio för Mac eller Visual Studio-kod på datorer och Mac-datorer.

## <a name="plan-and-design-with-back-end-services"></a>Planera och designa med backend-tjänster
 Microsoft tillhandahåller en mängd olika tjänster för att konfigurera en säker, skalbar och tillförlitlig backdel för din mobilapplikation. Du kan skapa serverlösa API:er och utlösare utan att behöva underhålla dina egna servrar. Använd push-meddelanden, offlinedatasynkronisering, autentisering och datatjänster för att skapa en omfattande och expansiv startslut som stöder en mängd olika scenarier för appkonstruktion. 
 
 Att bygga en back-end-tjänst har många krav. Du måste:
   - Begränsa datalagringen på enheten.
   - Synkronisera data mellan flera enheter.
   - Kontrollera att programmet körs utan nätverksanslutning.
   - Skicka meddelanden till användare.
   - Minska batteriförbrukningen.
   - Skapa en skalbar, tillförlitlig och säker serverdel.
   - Automatisera serverunderhåll.

## <a name="devops-and-continuous-monitoring"></a>DevOps och kontinuerlig övervakning
För att skapa mobila och molnfokuserade program, dra nytta av kontinuerlig integration (CI), kontinuerlig leverans (CD) och kontinuerlig övervakningstjänster för att producera inbyggda byggen för alla plattformar. Du kan köra ui automation tester på tusentals fysiska enheter, automatisera deras versioner via betatestkanaler, eller distribuera direkt till app butiker. Du kan också samla in användningsanalys, fel och undantagsinformation för deras program.

## <a name="additional-services"></a>Ytterligare tjänster
Många Azure-tjänster kan förbrukas beroende på typen och typen av program som byggs:
  - Azure Cognitive Search för att söka efter alla typer av interna data, med snabb frågor.
  - Azure Cognitive Services för att använda artificiell intelligens i ditt program.
  - Azure SignalR för att aktivera realtidskommunikation i ditt program.
